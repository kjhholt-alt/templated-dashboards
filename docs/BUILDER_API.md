# Builder API

The fluent builder is the recommended way to construct a dashboard
from Python. It is a thin layer over the Pydantic IR models in
`dashboards.ir`.

## Quick start

```python
from dashboards import Dashboard, render

dash = (
    Dashboard("War Room", subtitle="2026-05-04", theme="palantir")
    .section("Outreach", layout="grid")
        .kpi("Sends",   226, delta="+12", tone="good")
        .kpi("Replies",   1, delta="+1",  tone="good")
        .kpi("Errors",    4, tone="warn")
    .section("Fleet")
        .table(
            caption="Project health",
            headers=["Project", "Status", "Last Deploy"],
            rows=[
                ["operator-core", "GREEN", "2h ago"],
                ["ai-ops",        "RED",   "20d ago"],
            ],
        )
    .section("Events")
        .timeline([
            {"when": "2026-05-04 12:10", "title": "deploy", "tone": "good"},
            {"when": "2026-05-03 22:40", "title": "preview ERROR", "tone": "bad"},
        ])
    .section("Notes")
        .callout("ai-ops still red - codex lane", tone="bad")
        .callout("All status files updating", tone="good")
    .footer("generated by operator-core")
    .build()                 # -> dashboards.ir.Dashboard
)

print(render(dash, "html"))
```

## Methods

### Top level

* `Dashboard(title, *, subtitle=None, theme="palantir")` — start.
* `.section(title, *, subtitle=None, layout="stack")` — open a new
  section. Subsequent `.kpi/.table/...` calls add to the most-recent
  section.
* `.footer(text)` — set footer.
* `.build()` — return a Pydantic-validated `dashboards.ir.Dashboard`.
* `.to_dict()` — like `.build()` then `.model_dump(exclude_none=True)`.

### Components

All return `self` so calls chain.

* `.kpi(label, value, *, delta=None, tone="neutral")`
* `.status_card(project, *, status_url=None, inline_status=None)`
* `.table(*, headers, rows, caption=None)`
* `.timeline(events)` — `events` is an iterable of dicts with `when`,
  `title`, optional `detail`, optional `tone`.
* `.callout(text, *, tone="neutral", icon=None)`
* `.chart(kind, series, *, caption=None)` — `kind` is
  `"bar"`/`"line"`/`"sparkline"`; `series` is an iterable of
  `{"label": str, "points": [floats]}` dicts.

### Tones

`"good"`, `"warn"`, `"bad"`, `"neutral"`. Themes map these to colours.

## Errors

Calling a component method before `.section(...)` raises
`RuntimeError` — this catches "I forgot to open a section."

Validation happens in `.build()` via Pydantic. Bad inputs (missing
required fields, wrong types) raise `pydantic.ValidationError`.

## Why no end-of-section call?

Sections are bounded by the next `.section(...)` call (or `.build()`).
This keeps the chain shallow. If you need to programmatically build
sections from a list, just call `.section(...)` in a loop:

```python
b = Dashboard("Multi")
for proj in projects:
    b = b.section(proj.name)
    for k, v in proj.kpis.items():
        b = b.kpi(k, v)
return b.build()
```
